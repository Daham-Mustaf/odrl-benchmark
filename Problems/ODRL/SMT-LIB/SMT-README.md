# SMT-LIB Encoding — TPTP-ODRL Benchmark Suite

**Logic:** `UF` (uninterpreted functions, single sort) · **Solver:** Z3 ≥ 4.12

## Overview

Parallel SMT-LIB2 encoding of the TPTP-ODRL benchmarks. All 34 problems are self-contained `.smt2` files with axioms inlined (no include mechanism in SMT-LIB). Generated by `generate_smtlib.py` at the project root.

## Verdict Mapping

| TPTP (Vampire) | SMT-LIB (Z3) | ODRL Verdict |
|:---:|:---:|:---:|
| Theorem | `unsat` | Compatible or Conflict |
| CounterSatisfiable | `sat` | Unknown |

The negated conjecture pattern is identical: assert the negation of the claim, then `(check-sat)`. `unsat` = negation is impossible = claim holds.

## Translation from TPTP

### Sort System

TPTP is unsorted; SMT-LIB requires sorts. We use a single uninterpreted sort `Entity` for all terms (domain concepts, constraint names, operator names, operand names).

```smt2
(declare-sort Entity 0)
```

### Axiom Inlining

TPTP uses `include('Axioms/...')`. SMT-LIB has no include. The generator assembles files from modular components matching the four-layer architecture:

| TPTP Layer | SMT-LIB equivalent |
|---|---|
| `GEO000-0.ax` | `GEO_KB` constant (partOf + facts) |
| `DPV000-0.ax` | `DPV_KB` constant (subClassOf + DAG) |
| `LNG000-0.ax` | `LNG_KB` / `LNG_KB_FACTS` constants |
| `ODRL000-0.ax` | `LAYER1` constant |
| `GROUND000-1.ax` | Modular `GROUND_*` constants |

### Modular Grounding Rules

DPV-only problems don't declare `partOf`. In TPTP, unused predicates are harmless. In SMT-LIB, referencing undeclared functions is a parse error. The generator splits grounding rules:

| TPTP (single file) | SMT-LIB (per-domain) |
|---|---|
| `denotation_isAnyOf_taxonomic` | `GROUND_ISANYOF_TAX` |
| `denotation_isAnyOf_mereological` | `GROUND_ISANYOF_MEREO` |
| `denotation_isAllOf_taxonomic_only` | `GROUND_ISALLOF_TAX_ONLY` |
| `denotation_isNoneOf_taxonomic_only` | `GROUND_ISNONEOF_TAX_ONLY` |

### Conjecture Encoding

TPTP:
```tptp
fof(compatible, conjecture, ?[X]: (in_denotation(X,c1) & in_denotation(X,c2))).
```

SMT-LIB (negated + check-sat):
```smt2
(assert (not (exists ((x Entity))
    (and (in_denotation x c1) (in_denotation x c2)))))
(check-sat)
; unsat = Theorem = claim holds
```

### Per-Problem Grounding

Set operators with hard directions (isAllOf if, isNoneOf if, isAnyOf only-if) are encoded per-problem in both TPTP and SMT-LIB:

```smt2
; isAnyOf only-if (grounded disjunction)
(assert (forall ((x Entity))
    (=> (in_denotation x c1) (or (subClassOf x de) (subClassOf x en)))))
```

## Multi-Prover Results

All 34 problems: **100% verdict agreement** between Vampire and Z3.

| Metric | Vampire | Z3 |
|---|---:|---:|
| Fastest (unsat) | 0.002s | 0.02s |
| Slowest (sat) | 0.024s | 0.34s |
| Peak memory | 8 MB | ~30 MB |

Z3 is ~3–10× slower on these problems. Vampire's EPR specialization and AVATAR clause splitting give it an advantage. Z3 `sat` results take longer due to model construction.

## Generator Architecture

`generate_smtlib.py` is a single Python script (~800 lines) with:

- **KB constants** — GEO_KB, DPV_KB, LNG_KB, LNG_KB_FACTS
- **Layer constants** — LAYER1, GROUND_EQ, GROUND_ISA, etc.
- **Helper functions** — constraint(), conjecture_compatible(), isallof_if_taxonomic(), etc.
- **Problem catalog** — 34 `add()` calls, each specifying KBs, grounding rules, constraints, conjecture, expected results
- **Runner** — generates files, optionally runs Z3/CVC5, prints comparison table

To add a new problem: add an `add()` call to the catalog, then `python3 generate_smtlib.py --run`.

## Known Limitations

1. **No incremental solving.** Each file is standalone. No axiom sharing across problems.
2. **Single sort.** No type safety at the SMT level — type guards (`mereological`, `taxonomic`) are axiom-level, not sort-level.
3. **No `(get-model)`.** CounterSat results don't extract the satisfying model. Use Vampire's saturation output for model inspection.
