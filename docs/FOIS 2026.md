**Naming convention rules:**

| Part | Rule | Example |
|---|---|---|
| Prefix | `GRND` for all FOIS grounding problems | `GRND` |
| Category | `C`=consistency, `E`=entailment, `F`=conflict, `X`=cross-level, `D`=discrimination | `E` |
| Number | 3-digit zero-padded | `001` |
| Version | always `+1` for TPTP SZS | `+1` |
| Extension | `.p` for problems, `.ax` for axiom includes | `.p` |

---

**Three rules for keeping it clean:**

1. **Never mix** — `Problems/ODRL/` is constraint semantics, `Problems/DeonticOntology/` is deontic grounding. No cross-includes between the two trees.
2. **Each axiom gets its own `.ax` file** — `GRND-ALL.ax` just `include`s them. This way §6.1 can cite exact file names.
3. **Generator scripts are in `Generators/DeonticOntology/`** — separate from `Generators/` (existing VLDB scripts).
